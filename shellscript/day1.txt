1. New project GitHub pe push karna hai from scratch — full commands likho
ANS - 
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin <repo-url>
git push -u origin main

2.Difference between:
git add
git commit
git push
Ans -
git add - changes ko staging me le aana. ab ye file commit ke liye ready hai.
git commit - local repo me comment karke save karna means uska snapshot banati hai.
git push - local commit ko remote repository (github) pe bhejta hai.

3. Git Status kya batata hai?
Ans - git status shows working directory, staging area, and untracked files
Kaunse files modified hain
Kaunse files staging area me hain (git add ke baad)
Kaunse files untracked hain (new files)
Kaunsa branch active hai
Repo clean hai ya nahi (ek file change hui / ek new file add hui)

4.Galat commit ho gaya — kaise undo karoge?
Ans - I use reset for local changes and revert for already pushed commits 
(If editior open then enter ESC + :wq )

5. git pull vs git fetch difference?
Ans - git fetch downloads changes from remote but does not merge, while git pull is a combination of fetch and merge that updates the local branch directly
git pull - Remote repository (GitHub) se code fetch + merge karke local branch me le aata hai
git fetch - Sirf remote changes ko download karta hai lekin merge nahi karta.

Branch kaise create, switch aur delete karte hain?
Ans - I use git branch to create a branch, git checkout or git switch to move between branches, and git branch -d to delete branches. For creating and switching together, I use git checkout -b branch-name.
| Action          | Command                  |
| --------------- | ------------------------ |
| Create          | `git branch design`      |
| Switch          | `git checkout design`    |
| Create + switch | `git checkout -b design` |
| Rename main     | `git branch -M main`     |
| Delete          | `git branch -d design`   |
